# Re-Audit Comparison Design

## Status
Approved for implementation

## Route
`/workspaces/[workspaceId]/re-audit`

## Purpose
The anchor for every monthly review. Shows what was fixed, what regressed, and whether the tracked actions actually drove the score improvement. This is the number the HOD puts in a board meeting slide.

---

## Baseline & Comparison Selector

Two scan selectors side-by-side at the top of the page:

- **"Since scan:"** — dropdown listing all scan timestamps (newest to oldest), showing date, scan type, overall score, and issue summary per option
- **"vs scan:"** — same format, defaults to the next-oldest scan

A third control: **"Cycle comparison"** toggle. When on, auto-selects the two scans anchoring the current growth cycle using `growthCycles[workspaceId].auditId` (latest) and `.baselineSnapshotId`. The selectors become locked while active and a small label shows which cycle is loaded.

Quick-pick chip row below selectors:
- `"Last scan vs 30d ago"` — most common comparison
- `"This cycle vs baseline"` — pulls from active growth cycle
- `"vs oldest scan"` — longest view available

On mobile, stacked vertically.

---

## Score Comparison Card

Prominent top card showing:

| Field | Format |
|-------|--------|
| Score before | `72 · Watch` with the date |
| Score after | `81 · Healthy` with the date |
| Delta | `+9` in lime, with verdict label |
| Verdict | One short sentence |

The verdict label is auto-generated by `generateVerdictLabel(comparison)`:
- Score up + criticals resolved → positive framing
- Score up but new criticals → mixed warning
- Score down → direct regression copy

---

## Action-Attribution Panel

Only shows actions with `status: "Done"` AND evidence with `approvalStatus: "Approved"` where `completedAt` falls between the two selected scans.

| Action | Category | Owner | Completed | Score Before → After |
|--------|----------|-------|-----------|-----------------------|
| Publish FAQ schema on course pages | AEO | Anika Rao | May 22 | 72 → 78 · +6 category |
| Optimize MBA admissions title + H1 | Technical | Rohan Mehta | May 21 | 91 → 94 · +3 category |

The score before/after is the relevant `AuditCategory.score` for that action's `impactArea`. Category scores are pulled from the workspace data for the two scan dates — since workspace audit categories don't carry timestamps, the comparison is anchored to the scan context and mapped through the action's `source` and `sourceId` fields.

Below the table: summary line showing total completed/approved actions driving the improvement and any actions still in evidence review not yet counted.

## Category Delta Grid

Grid of all `AuditCategory` scores from the workspace, comparing the two scans:

| Category | Before | After | Delta | Top Issue |
|----------|--------|-------|-------|-----------|
| Title Tags | 84 | 91 | +7 | Duplicate programme titles |
| Schema | 72 | 81 | +9 | Course schema incomplete |
| Content Quality | 79 | 74 | -5 | Missing admissions intent |

Each cell: category name, previous score, new score, signed delta with color coding, top issue label with severity chip.

Regressed categories (delta < 0) get an orange border to flag them immediately.

Only categories present in both scans are shown.

## Chain Comparison Mode

The two scan selectors naturally support comparing any two scans in the list — not just consecutive ones. Users can pick any date range.

---

## AI Draft Narrative

A text block at the bottom titled **"AI Draft Narrative"** — a 3-4 sentence paragraph summarizing the comparison in plain English, written for direct inclusion in the client report.

Has a **"Copy to clipboard"** button.

Generated by `generateReAuditNarrative(comparison, attributedActions, categoryDeltas)` — pure function, no live API call.

Future: editor-in-place with manager approval before publishing to client portal.

## Helpers (packages/shared/src/rankflow-helpers.ts)

### Existing
- `compareScans(current, previous)` — already exists, used as-is

### New
- `generateVerdictLabel(comparison: ScanComparison)` → `string` — picks appropriate 1-line verdict template
- `attributeActionsToScoreChange(actions: ActionItem[], auditCategories: AuditCategory[], scanBefore: ScanSnapshot, scanAfter: ScanSnapshot)` → `AttributedAction[]` — maps actions with approved evidence and `completedAt` in range to category score changes
- `generateReAuditNarrative(comparison: ScanComparison, attributedActions: AttributedAction[], categoryDeltas: AuditCategoryDelta[])` → `string` — assembles template-based narrative paragraph

### New types
```typescript
interface AttributedAction {
  action: ActionItem;
  category: AuditCategory;
  scoreBefore: number;
  scoreAfter: number;
}

interface AuditCategoryDelta {
  category: AuditCategory;
  scoreBefore: number;
  scoreAfter: number;
  delta: number;
  topIssue: string;
}
```
